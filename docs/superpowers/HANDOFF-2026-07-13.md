## Qué es este proyecto

`C:\Users\Jonathan\Desktop\fixlab-leads\` — app Flask que scrapea leads de Google Maps
(`prospector.py`, sin tocar) hacia 3 tabs de un Google Sheet (Repuestos/Fundas/Telefonos)
para outreach mayorista de FixLab Store en Balvanera/Once, CABA.

Esta sesión agregó un **generador de rutas comerciales** encima: lee esos leads,
los geocodifica, y arma lotes de links de Google Maps (~40 direcciones, partidas en
sub-lotes de ≤9 paradas encadenadas) para mandar al vendedor por WhatsApp.

- Repo git local, sin remoto. Todo en `master`, working tree limpio.
- Se corre con el acceso directo `FixLab Leads.lnk` del escritorio (usa `launch.bat`),
  o `python app.py` desde la carpeta del proyecto. Abrís `http://localhost:5000/`
  y la pestaña **Rutas** te lleva a `/rutas/`.
- **74 tests, todos verdes**: `pytest -v` desde la carpeta del proyecto.
- Spec original: `docs/superpowers/specs/2026-07-13-rutas-comerciales-design.md`
- Plan de implementación: `docs/superpowers/plans/2026-07-13-rutas-comerciales.md`

## Arquitectura (6 módulos nuevos, integrados a `app.py` existente)

- `routes_db.py` — schema SQLite (`leads_routes.db`, gitignored) + toda la capa de datos.
- `routes_geocoding.py` — cadena de fallback: Maps_URL → Nominatim por dirección → por nombre.
- `routes_algorithm.py` — haversine, selección de N más cercanos, nearest-neighbor, chunking, link de Maps.
- `routes_sheet_sync.py` — lee las 3 tabs del Sheet (solo lectura, nunca escribe), reusa el OAuth de `prospector.py`.
- `routes_batch.py` — orquesta todo: `generate_lote()`.
- `routes_app.py` — Blueprint Flask, todas las páginas de `/rutas/*`.

`app.py` solo tiene 6 líneas agregadas (import + `init_db()` + `register_blueprint`).

## Bugs encontrados y arreglados esta sesión (todos commiteados)

1. Abreviaturas argentinas que Nominatim no reconoce (Cnel.→Coronel, Gral.→General, Cdad.→Ciudad, etc.)
2. Sufijo redundante ", Buenos Aires, Argentina" cuando la dirección ya lo mencionaba — rompía el parser.
3. Mismo problema pero con "Argentina" sin "Buenos Aires" (ej. "Avellaneda, Argentina").
4. Código postal extendido (formato CPA de 8 caracteres) pegado sin coma antes de "Ciudad"/"Provincia".
5. Bounding box de seguridad (AMBA) en `nominatim_geocode` — rechaza cualquier resultado fuera de zona (evita falsos positivos geográficos).
6. **Bug importante**: `direccion = "-"` (placeholder de "sin dirección") se trataba como dirección real por el chequeo de truthiness de Python. Nominatim le devolvía la misma coincidencia genérica a 183 leads distintos → todos con las mismas coordenadas exactas. Esto se hizo visible cuando el picker de "nombre de negocio como origen" (ver abajo) eligió un lead corrupto como origen y el algoritmo de "más cercanos" agrupó a los otros 182 (distancia 0). Ya arreglado y los 183 reprocesados (17 recuperados por nombre, 166 quedaron fallido correctamente).

## Pendiente / lo que sigue (nada de esto está implementado, son ideas conversadas)

- **Google Maps/Apify para los 728 fallidos**: se discutió (sin implementar, usuario pidió no actuar todavía) reusar el scraper Apify existente (`prospector.py`) para buscar por nombre de negocio directo en Google Maps en vez de depender de Nominatim/OSM (que no indexa locales chicos). Costo estimado: ~$0.002/resultado (~$1-2 para los 728). Alternativa más precisa pero con más setup: Google Places API (Place Details) usando los `Place_ID` que ya tenemos guardados en varios cientos de leads — requiere API key + cuenta de Google Cloud con billing, cosa que se evitó deliberadamente al principio del proyecto para no generar costo recurrente.
- **CRM sobre las rutas** (fase 2, explícitamente diferida por el usuario): vista consolidada de las 3 categorías, filtros, tracking de outreach (contactado → respondió → convertido). Decisión pendiente: extender `leads_cache` en SQLite vs seguir usando las columnas manuales `Tipo_contacto`/`Estado` del Sheet — no duplicar ambas fuentes de verdad.
- **Estado de sincronización**: el usuario mencionó querer ver algún tipo de info/historial de las corridas de sync — no diseñado todavía, solo mencionado como intención futura junto con el CRM.
